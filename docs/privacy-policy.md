# TumorBoard AI 隐私政策与使用同意书 v1.1

**生效日期**:2026-05-27 · **版本**:v1.1 · **政策快照哈希**:`hash(this_doc) → user_consents.policy_version`

> **v1.1 变更摘要**:ASR 链路改为「火山引擎豆包音频理解 API」,**原音频将上传到火山自有云**(不再仅在自部署服务器内处理)。所有 v1.0 同意记录已失效,需重新签署。详见 §2 和 §3。

---

## 1. 工具定位

TumorBoard AI(以下简称"本工具")是中国肿瘤多学科会诊(MDT)的医生**个人辅助整理工具**,**不**是医疗器械,**不**接入任何医院 HIS/EMR/PACS 内网,**不**用于直接的临床诊断或治疗决策。

本工具的所有产出(病例摘要、TNM 分期、多学科意见、检查/治疗建议、患者反馈话术)均带"AI 辅助生成,需主治医师复核"水印,**最终临床决策必须由您本人作为执业医师独立做出**。

## 2. 您必须确认的事项

继续使用本工具前,您必须确认理解并同意:

- [ ] 我承诺**不在本工具中录入或上传**任何可识别患者身份的真实姓名。所有患者请使用**化名/代号/编号**(如 "P-2026-001"、"老李")。
- [ ] 我承诺在拍照前**遮挡**病历/检验单/影像报告上的:患者姓名、身份证号、手机号、家庭住址、病历号。本工具会对 OCR 出的文本做兜底脱敏,但**像素层面的 PII 无法脱敏**。
- [ ] 我承诺**不依赖**本工具的输出做单独的临床决策,所有建议在落实到患者前必须经我本人复核。
- [ ] 我承诺**不向第三方**(包括家属、患者本人)分享未经我审核的 AI 原始输出。
- [ ] 我理解 MDT 录音(**原音频**)会被上传到**火山引擎豆包音频理解 API**(由字节跳动旗下火山引擎运营,服务条款约定不用于训练、不向第三方分享)进行转写与说话人分离;返回的**脱敏文本**才会传给云端 LLM 做综合分析。
- [ ] 我承诺**仅在征得患者及参会医生口头同意录音后**才点击"开始录音"(MDT 会议开场的标准告知)。

## 3. 数据流与存储

### 3.1 在本工具内流转的数据类别

| 数据 | 来源 | 存储位置 | 保留期 |
|---|---|---|---|
| 原始病历图片 / PDF | 您上传 | MinIO(自部署,加密 bucket)+ **送火山引擎 OCR API 做识别(不留存于火山侧)** | **30 天**后归档,**365 天**后彻底删除 |
| 原始 MDT 录音 | 您录制 | MinIO(自部署,加密 bucket)+ **送火山引擎豆包音频理解 API 做转写(不留存于火山侧,不用于训练)** | **30 天**后归档,**365 天**后彻底删除 |
| OCR 后的文本 | 火山 OCR API 返回 | PostgreSQL JSONB | 同会话生命周期 |
| ASR 后的转写 | 火山豆包音频理解 API 返回 | PostgreSQL JSONB | 同会话生命周期 |
| AI 综合输出(摘要/TNM/意见/建议) | LLM 云端调用 | PostgreSQL | 同会话生命周期 |
| 您的医生身份(姓名/医院/科室) | 注册时您填写 | PostgreSQL,加密 | 直至您注销账号 |
| 审计日志 | 自动记录 | PostgreSQL | 365 天 |

### 3.2 数据不流向哪里

- **绝不接入医院 HIS/EMR/PACS** — 无内网接口,无医院数据交换协议。
- **绝不向第三方分享**任何会话数据 — 包括其他医院、其他医生、保险公司、广告平台等。
- **绝不用于训练任何模型** — 您的数据不会被用于改进本工具或任何 AI 模型的训练(火山引擎企业协议同样要求)。
- **下游 LLM(综合分析)不接收**:患者真实姓名(您已遮挡)、原始图片字节、原始音频字节;只接收 §3.3 中列出的脱敏文本。

### 3.3 云端 API 调用边界

本工具会调用两类云端 API,边界如下:

**A. 火山引擎多模态 API(OCR + 音频理解)**
- 接收:**原始图片字节** / **原始音频字节**(无法在像素/采样层脱敏 — 由您拍照前遮挡 + MDT 开场告知兜底)
- 返回:识别后的文本 / 转写后的分段文本
- 火山方:不留存原始字节,不分享给第三方,不用于训练(由企业服务条款约束)
- 我们方:返回的文本经 PII 脱敏后才进入下游 LLM 链路

**B. 下游 LLM(豆包/通义/Kimi/Claude,综合分析)**
- 仅接收以下脱敏后**文本**:
  - 火山 OCR 识别的**文本**(您手机号、身份证、邮箱、姓名标签后的字符已替换为 `<PHONE_1>` 等占位符,见 §4)
  - 火山音频理解返回的**转写文本**(同样脱敏)
  - AI 之前已生成的结构化字段(摘要/TNM/意见)
- **不接收**:任何图片字节、任何音频字节、任何含真实姓名的文件名(系统会重命名为内部 ID)

## 4. PII 脱敏机制(技术细节)

本工具在 OCR/ASR 文本进入 LLM 前会做以下处理(`utils/pii_scrubber.py`):

| 数据类型 | 正则识别 | 替换为 |
|---|---|---|
| 手机号 | `1[3-9]\d{9}` | `<PHONE_N>` |
| 身份证号 | 18 位标准格式 | `<ID_N>` |
| 银行卡号 | 16-19 位 | `<BANKCARD_N>` |
| 邮箱 | RFC 5322 简化 | `<EMAIL_N>` |
| "姓名:XXX" "患者:XXX" 标签 | 标签后 2-4 字 | `<NAME_N>` |
| 病历号 / 住院号 / 门诊号 | 6-16 位字母数字 | `<MRN_N>` |
| 详细住址 | 省市区路号格式 | `<ADDR_N>` |

**关键安全保证**:
- 占位符映射 `<NAME_1> → 张三` 只在**单次 LLM 调用闭包内**存在
- LLM 返回后立即在内存中还原占位符,**映射立即销毁**
- 1000 次并发调用,映射隔离 0 串台(`tests/test_pii_concurrent.py` 自动化验证)
- 映射**永不**进入日志、Redis、PostgreSQL、MinIO 或任何持久层

**限制**(您需要知道):
- 像素层面的 PII(原图像里的姓名签字)不会被本机制脱敏 — 上传前您必须自行遮挡
- 启发式姓名识别仅命中带"姓名:""患者:"标签的情况,白板上随手写的名字识别不出
- 罕见姓氏 / 外籍患者姓名识别可能漏召

## 5. 您的权利

- **查阅**:您可在 `/cases` 页面查看您创建的所有会话,以及它们对应的所有上传文件。
- **真删**:点击"删除会话"会**同时删除** MinIO 中的所有文件和 PostgreSQL 中的所有记录(包括审计日志中除"删除事件"以外的部分)。**删除是不可恢复的物理删除,不是软删除**。
- **导出**:Word/PDF/PPT/微信卡片格式导出,带水印,您本人保管。
- **撤回同意**:您可在设置页注销账号,系统将删除您的所有会话(同上"真删")。

---

## 6. 安全措施

- 所有 PostgreSQL 字段含 PII 部分使用 pgcrypto 加密 at-rest
- 所有 MinIO bucket 设置为私有,Lifecycle 30 天归档 / 365 天彻底删除
- 所有 HTTP 接口走 HTTPS / SSE
- 审计日志(`audit_logs` 表)记录每次创建/编辑/导出/删除事件,365 天保留
- 含 PII 的字段从不进入应用日志(`utils/logger.py` 默认 JSONL,raw_text 字段被显式禁止)

---

## 7. 合规边界

本工具按以下定位设计与运营:
- **不属于医疗器械**(无诊断/治疗决策算法的直接输出)
- **不接入院内系统**(避免触发医院信息安全等保+医疗器械备案)
- **数据归您本人**(自部署或单医生 SaaS 模式,无多医院共享)

如您所在医院希望部署机构版,需要联系运营团队签署:
- 数据处理协议(DPA)
- 跨境数据传输评估(若使用境外 LLM provider)
- 医院信息安全等保 2.0 三级评估配套文件

---

## 8. 紧急情况

若您怀疑本工具的输出含错误信息(尤其是 TNM 分期 / 治疗建议):
1. **不要**依据该输出做临床决策
2. 在 review 页点击"标记问题"反馈
3. 必要时通过指南/教科书复核

若您怀疑您的账户被他人冒用或数据泄露:
1. 立即在设置页修改密码
2. 在审计页查看最近 30 天操作记录
3. 联系运营团队(邮箱见 README.md)
