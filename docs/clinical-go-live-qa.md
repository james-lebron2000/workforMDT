# 临床上场前 10 分钟自检 Checklist

> **用法**:开门诊前一天/当天早上,拿你下临床要用的那台手机 + 那个网络,照下面走一遍。任何一项不通过就回来找开发,别凑合上场。

---

## A. 前置 5 项(2 分钟)

- [ ] **A1** 手机系统:iOS ≥ 16.4 或 Android Chrome ≥ 110。低于这个版本 Wake Lock / MediaRecorder 兼容性不保。
- [ ] **A2** 关掉省电模式(iOS:设置 → 电池 → 低电量模式 关;Android:同名设置 关)。省电模式会暂停后台 MediaRecorder。
- [ ] **A3** 手机存储剩余 ≥ 2GB(用于 IndexedDB 缓存最长 30 分钟录音 ~20 片)。
- [ ] **A4** 本院 WiFi 可用,且能打开 `https://<MDT 服务地址>/diagnostics`(后端给你的地址)。
- [ ] **A5** **不要**在微信里点链接打开 — 微信内置浏览器对 MediaRecorder 兼容性差。
  - iPhone:复制链接到 Safari 打开。
  - 安卓:复制链接到 Chrome 打开。

---

## B. /diagnostics 自检(1 分钟)

打开 `/diagnostics`,等 5 秒看结果:

- [ ] **B1** 7 项全绿(✅)。任何黄/红按页面提示修。
- [ ] **B2** 重点确认这 4 项必须绿:
  - 浏览器与设备(若黄,微信内核 → 换 Safari/Chrome)
  - 后端服务 /health/deep(若红 → 联系后端管理员,所有依赖不通就别上场)
  - 麦克风权限(若红 → 设置里允许;若 iOS 一直拒绝,卸载浏览器重装一遍权限弹窗才会再出)
  - MediaRecorder + 编码格式(若红 → 设备/系统过旧,换台手机)
- [ ] **B3** Wake Lock 黄了不致命,但你要手动:设置 → 显示与亮度 → 自动锁定 → **永不**。会议结束记得改回去。
- [ ] **B4** IndexedDB 红 → 你在无痕模式,退出无痕。

> 截一张 /diagnostics 全绿截图,这是上场凭证。

---

## C. 主流程冒烟(5 分钟)

用一个**测试病例代号**(如 `TEST-001`,不要用真实患者代号),把 4 步走一遍:

### C1. 新建病例 + 上传资料

- [ ] 打开 `/cases` → 点 "+ 新建 MDT" → 输 `TEST-001` + 性别 + 年龄段
- [ ] 进入上传页 → 拍 **3 张**任意化验单/病历照(可以拍现成纸张就行)
- [ ] 看每张照片下方在 30 秒内显示"OCR 完成"
- [ ] **门禁验证**:在同意书未签的情况下点上传,应该被拒/弹同意书。已签则正常。

### C2. 录 1 分钟患者诉求 + 生成病史摘要

- [ ] 录音 60 秒 → 点"完成并转写"
- [ ] 看到"编码:audio/webm 或 audio/mp4"显示
- [ ] 等 30~60 秒看到转写结果
- [ ] 点"生成病史摘要"等结果出现
- [ ] **红线 1**:摘要里有 timeline,每条带原文引用 — 如果是 AI 编的,timeline 会空或引用是模糊"根据上述"之类
- [ ] 点"病史已与患者核对" → status 升到 `summary_confirmed`(不点这步,下面 C5 的会前 PDF 导不出来)

### C3. MDT 录音 — 30 分钟锁屏压测(**最关键的一项**)

> 这一步必须真做一次。会议中途锁屏导致丢音是最常见的临床事故。

- [ ] 切到"会议录音"Tab → 点开始
- [ ] 看到红点闪烁 + "屏幕常亮中"(Wake Lock 成功)
- [ ] 主动按电源键**息屏** 5 分钟,期间随便讲话(对着话筒)
- [ ] 解锁屏 → 看时间应继续走、分片数应继续涨
- [ ] **断网 30 秒压测**(开飞行模式 30 秒再关):
  - [ ] 分片状态从"上传中"→ "失败" → 你看到"重传失败片"按钮
  - [ ] 关掉飞行模式后,网络恢复时片应该被自动重试,或你点按钮重传
- [ ] **后台压测**:把浏览器切到后台 2 分钟 → 切回来,录音应仍在录
- [ ] 录满 30 分钟点结束 → "完成并转写"
- [ ] **红线 2**:finalize 后端会用 ffmpeg 转 mp3 — 如果报"音频转码失败",别接着用,联系后端

> 不通过的话:很可能是省电模式没关 / 微信内核 / iOS < 16.4。

### C4. AI 分析 + 医生确认

- [ ] 点"开始 AI 分析" → SSE 进度条应该走起来,30 分钟录音通常 3~5 分钟出结果
- [ ] **断网 30 秒压测 SSE**:分析中拔网 30 秒 → 应该自动重连(看到"已重连"或进度继续)
- [ ] 进 `/cases/[id]/review` 看 8 张卡片
- [ ] **红线 3**:检查 6 科是不是都有显式标签 — 没说话的科室必须显示"本次讨论未明确记录,建议补充"。如果某科是空白或被 AI 编了内容 → 红牌
- [ ] **红线 4**:TNM 卡片必须有 `basis` 文字 + 至少 1 条 evidence 原文。没 basis → 红牌,联系开发
- [ ] **红线 5**:点开"给患者的话术"全文搜索这 7 个词必须 0 命中:`治愈 / 一定 / 保证 / 百分百 / 肯定治好 / 包治 / 永不复发`
  - 命中:点该卡片"重新生成",再不行手改

### C5. 导出

- [ ] 点"导出报告" → 勾选水印声明
- [ ] 导 Word — 打开 docx,确认第一页 + 页脚都有"AI 辅助生成,需主治医师复核"
- [ ] 导 PDF — 每页底部应有水印
- [ ] 导微信卡片 — 复制后 7 禁词应该 0 命中(再搜一遍)
- [ ] 回到 review 页点"导出会前摘要 PDF" — 应该能导(因为 C2 已确认)
- [ ] **红线 6**:打开会前 PDF,确认**不包含** TNM/科室意见/治疗建议(会前不该有这些)

---

## D. 收尾 2 分钟

- [ ] 在病例列表把测试病例 `TEST-001` **真删** — 点删除,确认提示"将同步删除 N 个文件"
- [ ] 删除后刷新页面,病例列表里没了
- [ ] **真删验证**(可选,需后端配合):MinIO 后台搜该 session,音频/图片应物理消失;DB `audit_logs` 里有 `session.delete` 一条
- [ ] 把 /diagnostics 截图 + 这份 checklist 勾完发给开发存档

---

## 如果中途任何一步红了

**不要凑合上场**。临床场景出错比延期上线代价大 10 倍。

按这个顺序求助:
1. 浏览器/系统/网络问题 → 自查(切 Safari/Chrome,换 WiFi)
2. 后端 /health/deep 不通 → 找后端管理员
3. AI 输出红线问题(假科室意见/TNM 无 basis/话术承诺词)→ **立即停用**,找开发 hotfix prompt 再上场
4. 录音丢片/转码失败 → 找开发查 ffmpeg + IndexedDB

---

## 临床中现场需要

- [ ] 手机充电 ≥ 70%(30 分钟录音 + 屏幕常亮约耗 25%)
- [ ] 准备一个**备份录音**:开本机自带录音 app 同时录(以防主流程任何一环失败,还能让人工后期整理)
- [ ] 知道 `/diagnostics` 的快捷入口位置 — 现场卡了立刻打开看是哪项黄了
